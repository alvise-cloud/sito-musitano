OBIETTIVO

Pubblicare la versione Flask completa del sito, quindi con:
- sito pubblico
- admin
- dashboard
- prenotazioni salvate nel database
- HTTPS senza avviso "Non sicuro"


STATO ATTUALE

GitHub Pages puo pubblicare solo file statici. Quindi non puo eseguire:
- app.py
- database
- login admin
- dashboard
- salvataggio prenotazioni lato server

Per avere l'admin online serve un hosting Python/Flask.


PROBLEMA "NON SICURO"

Il dominio non deve puntare contemporaneamente a GitHub Pages e SiteGround.

Record corretti se si resta su GitHub Pages statico:
- A profmusitanofertility.it -> 185.199.108.153
- A profmusitanofertility.it -> 185.199.109.153
- A profmusitanofertility.it -> 185.199.110.153
- A profmusitanofertility.it -> 185.199.111.153
- CNAME www -> alvise-cloud.github.io

Record da rimuovere se presente:
- A profmusitanofertility.it -> 35.214.164.248

Quel record SiteGround crea conflitto con GitHub Pages e puo causare errore HTTPS/certificato.


PROBLEMA ADMIN

Se si vuole l'admin, il dominio dovra puntare all'hosting Flask, non a GitHub Pages.

Su hosting Flask, i DNS finali saranno quelli forniti dall'hosting scelto.
Quindi i record GitHub Pages andranno sostituiti con quelli del nuovo hosting.


VARIABILI SERVER NECESSARIE

ADMIN_USERNAME=...
ADMIN_PASSWORD=...
SECRET_KEY=una_chiave_lunga_casuale
SESSION_COOKIE_SECURE=1
FLASK_DEBUG=0

Opzionale:
DATABASE_URL=...

Se DATABASE_URL non viene impostato, l'app usa SQLite locale.


COMANDI HOSTING FLASK

Build:
pip install -r requirements.txt

Start:
gunicorn app:app


CONTROLLO FINALE

Dopo la pubblicazione Flask:
1. aprire https://profmusitanofertility.it
2. verificare assenza avviso "Non sicuro"
3. aprire /admin
4. fare login
5. creare una prenotazione di prova
6. verificare che compaia in dashboard
7. cancellare la prova
